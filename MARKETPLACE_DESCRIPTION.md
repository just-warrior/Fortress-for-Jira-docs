# Marketplace Description

**Last Updated:** June 10, 2026

## App Name

Jira Fortress for Jira

## Short Description

Turn Jira issue assignment into a lightweight fortress battle, enabled only for administrator-approved projects.

## Long Description

Jira Fortress for Jira adds a Forge-native battle panel to Jira issues. Teams can open a turn-based fortress duel, take shots, move tanks, react with emoji, and let the final battle result determine the issue assignee.

Administrators stay in control with a project key allowlist. By default, the app is disabled everywhere. Add only the Jira projects where your team wants to use Jira Fortress, and the issue panel will show a disabled message everywhere else.

The app is hosted on Atlassian Forge and does not use external app servers, external databases, or third-party analytics tools.

## Highlights

- Forge-native Jira Cloud app
- Jira administrator project allowlist
- Issue panel battle launcher
- Modal battle arena for two players
- Turn-based movement, aiming, firing, emoji, timeout, and forfeit flow
- Server-validated game actions for multiplayer sync
- Battle result assignment update
- Jira comments for battle events where applicable
- No external servers or third-party analytics

## Suggested Marketplace Summary

Make issue assignment more engaging with a controlled, project-scoped fortress battle inside Jira.

## Suggested More Details Copy

Jira Fortress for Jira gives teams a playful way to decide who takes an issue. Open the Assignee Battle panel, join the battle, and compete in a lightweight turn-based arena. When the battle ends, Jira Fortress attempts to update the issue assignee based on the result.

Admins can enable the app only where it belongs. Use the Configure page to add approved Jira project keys. Empty configuration means the app is disabled for every project.
